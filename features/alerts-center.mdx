## Overview

The Alerts Center is your proactive monitoring system. Instead of discovering problems after they impact results, the alert engine continuously evaluates your sales data and surfaces issues requiring attention — along with specific recommended actions.

<Note>
  The Alerts Center transforms reactive management into proactive problem-solving by detecting issues early and recommending next steps.
</Note>

## Business Value

<Check>**Early Warning System** — Detect problems before they become crises</Check>
<Check>**Prioritized Actions** — Focus on high-impact issues first based on severity</Check>
<Check>**Contextual Guidance** — Each alert includes specific entity details and suggested actions</Check>
<Check>**Team Accountability** — Track which alerts have been reviewed and addressed</Check>

## Alert Severity Levels

All alerts are classified into four severity tiers:

  <Card title="🔴 Critical" icon="circle-exclamation" color="#E84855">

  <Card title="🟡 Medium" icon="triangle-exclamation" color="#FFB347">
    **Action needed this week** — Significant but not urgent

    - Customer concentration risk

  <Card title="🔵 Low" icon="circle-info" color="#3B82F6">
    **Monitor closely** — Informational awareness

    - SKU rotation slowing

  <Card title="🟢 Opportunity" icon="lightbulb" color="#00C49F">
    **Growth potential detected** — Positive action recommended

</CardGroup>

## Alert Summary Dashboard

The top of the page displays alert counts by severity:

- **Critical (🔴):** Number requiring immediate action
- **Medium (🟡):** Number needing attention this week
- **Low (🔵):** Informational items to monitor
- **Opportunities (🟢):** Growth actions to consider
- **Total Alerts:** All active alerts for current filters

<Tip>
  If you have 5+ critical alerts, prioritize addressing those before reviewing medium/low items. Focus resources where impact is highest.
</Tip>

## Alert Filtering

Refine the alert list using three filter dimensions:

### Severity Filter
Show only specific severity levels:
- Select multiple severities
- Useful for daily triage (review critical only)
- Or weekly planning (review all medium/low)

### Entity Type Filter
Focus on specific alert categories:
- **Vendedor:** Salesperson performance alerts
- **Cliente:** Customer relationship alerts  
- **Producto:** Product inventory/sales alerts
- **Cartera:** Portfolio health alerts

### Hide Reviewed Toggle
Optionally hide alerts you've already reviewed:
- Keeps list focused on new items
- Reviewed status persists during your session
- Resets on page refresh

<Note>
  Filter combinations are powerful — e.g., "Show only Critical Customer alerts" to focus your customer rescue efforts.
</Note>

## Alert Details

Each alert card displays:

### Alert Header
- **Alert ID:** Unique identifier (e.g., ALT001, ALT002)
- **Alert Name:** Descriptive title (e.g., "Vendedor bajo objetivo")
- **Severity Badge:** Visual classification with color coding

### Entity Information  
- **Entity Name:** Specific salesperson, customer, or product affected
- **Entity Type:** Category (Vendedor / Cliente / Producto)
- **Detailed Description:** Specific metrics triggering the alert

Example: "Cliente XYZ (categoria A) — No registra compras en los últimos 35 días, cuando su frecuencia habitual es 15 días"

### Suggested Action
A specific recommended next step to address the alert:

- "Agendar visita comercial esta semana"
- "Revisar causas de la caída con el vendedor asignado"  
- "Aumentar stock para evitar quiebre"
- "Considerar promoción para impulsar rotación"

### Review Status
Indicates if the alert has been marked as reviewed in your session

## Built-in Alert Rules

The system includes 7 pre-configured alert rules:

<AccordionGroup>
  <Accordion title="ALT001 — Salesperson Below Quota" icon="user-xmark">
    **Trigger:** Salesperson objective fulfillment &lt;70%
    
    **Severity:** 🔴 Critical
    
    **Detection Logic:**
    ```python
    kv_df = kpi_por_vendedor(df, obj_df, df_ant)
    alertas = kv_df[kv_df['cumplimiento_pct'] < 0.70]
    ```
    
    **Suggested Action:** "Agendar sesión de coaching para identificar obstáculos y definir plan de acción"
  </Accordion>
  
  <Accordion title="ALT002 — Customer Inactivity Risk" icon="user-clock">
    **Trigger:** Customer hasn't purchased in 30+ days
    
    **Severity:** 🟠 High
    
    **Detection Logic:**
    ```python
    kcli = kpi_clientes(df, cli_df, df_ant, fecha_hasta)
    alertas = kcli[kcli['dias_desde_ultima_compra'] > 30]
    ```
    
    **Suggested Action:** "Contactar al cliente para entender causas de inactividad y ofrecer incentivos"
  </Accordion>
  
  <Accordion title="ALT003 — Product Sales Decline" icon="box-open">
    **Trigger:** Product sales down >30% month-over-month
    
    **Severity:** 🟠 High
    
    **Detection Logic:**
    ```python

    alertas = kp[kp['vs_periodo_anterior'] < -0.30]
    ```
    
    **Suggested Action:** "Investigar causas (precio, competencia, calidad) y ajustar estrategia comercial"
  </Accordion>
  
  <Accordion title="ALT004 — Portfolio Concentration Risk" icon="chart-pie">
    **Trigger:** Gini index >0.60 indicating excessive customer concentration
    
    **Severity:** 🟡 Medium
    
    **Detection Logic:**
    ```python
    conc = get_concentracion_pareto(kcli, 'importe_total')
    if conc['indice_gini'] > 0.60:
        # Generate alert
    ```

    **Suggested Action:** "Diversificar cartera para reducir dependencia de pocos clientes clave"

  <Accordion title="ALT005 — Margin Below Threshold" icon="percent">

    **Severity:** 🔴 Critical
    
    **Detection Logic:**
    ```python

    if km['margen_pct'] < MARGEN_MINIMO:  # e.g., 15%
    ```

    **Suggested Action:** "Revisar estructura de costos y pricing; considerar ajustes de precios"

    **Detection Logic:**

    ```python
        # Generate alert
    ```

    **Suggested Action:** "Incrementar frecuencia de visitas para reactivar clientes inactivos"

  <Accordion title="ALT007 — Upsell Opportunity" icon="arrow-trend-up">

## How Alerts Are Generated

The alert engine runs automatically when you load the page:

<Steps>
  <Step title="Load Sales Data">
    System loads filtered transaction data based on sidebar selections
  </Step>
  
  <Step title="Calculate KPIs">
    Runs all KPI engines:
    - `kpi_por_vendedor()` for salesperson metrics
    - `kpi_clientes()` for customer analytics
    - `kpi_productos()` for product performance
    - `kpi_cobertura_clientes()` for portfolio coverage
    - `kpi_margen()` for profitability
  </Step>
  
  <Step title="Evaluate Alert Rules">
    Each alert rule (ALT001-ALT007) checks if trigger conditions are met
  </Step>
  
  <Step title="Rank by Severity">
    Alerts are sorted: Critical → High → Medium → Low → Opportunity
  </Step>
  
  <Step title="Display in UI">
    Alerts shown with full context, entity details, and suggested actions
  </Step>
</Steps>

**Core function:** `get_alertas_activas(df, df_ant, cli_df, prod_df, obj_df, fecha_hasta)` in `src/alerts_engine.py`

Returns a DataFrame with columns:
- `id_alerta`: Alert rule ID
- `nombre`: Alert display name
- `severidad`: critical / alta / media / baja / info
- `entidad_tipo`: Vendedor / Cliente / Producto / Cartera
- `entidad_id`: ID of affected entity
- `entidad_nombre`: Display name of entity
- `descripcion_detalle`: Specific metrics causing alert
- `accion_sugerida`: Recommended next step

## Managing Alerts

### Mark as Reviewed

Click the **"✅ Marcar revisada"** button to:
- Flag the alert as reviewed in your session
- Remove it from the active list (if "Hide Reviewed" is enabled)
- Track that you've acknowledged and taken action

<Note>
  Reviewed status is session-based only. It resets when you refresh the page or start a new session. For production use, consider integrating with a task management system for permanent tracking.
</Note>

### Clear Review History

Click **"🔄 Limpiar historial de revisadas"** to:
- Reset all reviewed flags
- Show all alerts again
- Useful for weekly review meetings

## Using Alerts for Team Management

### Daily Stand-up
1. **Filter to Critical** — Review red alerts only
2. **Assign owners** — Delegate each alert to responsible person
3. **Set deadlines** — Critical = today, High = this week
4. **Mark reviewed** — Track what's been addressed

### Weekly Business Review
1. **Review all severities** — Complete picture of business health
2. **Identify patterns** — Are same issues recurring?
3. **Measure progress** — Are alert counts decreasing over time?
4. **Adjust strategies** — Change approaches if alerts persist

### Monthly Planning
1. **Export alerts to CSV** — Analyze trends over time
2. **Set preventive goals** — E.g., "Reduce customer inactivity alerts by 50%"
3. **Measure team response** — How quickly are alerts being addressed?

## Customizing Alert Rules

While the system includes 7 standard rules, you can customize:

### Adjust Thresholds

Edit threshold values in alert engine to match your business:

```python

# Example: Extend customer inactivity window  
ALT002_DIAS_INACTIVIDAD = 45  # Alert after 45 days instead of 30
```
