Get your sales intelligence dashboard up and running in under 5 minutes. This guide walks you through cloning the repository, installing dependencies, generating mock data, and launching the application.

## Prerequisites

- **Python 3.10 or higher** (3.10-3.12 recommended)
- **Git** installed on your system
- Basic familiarity with command line/terminal

## Quick Start Steps

<Steps>
  <Step title="Clone the repository">
    Open your terminal and clone the project:

    ```bash
    git clone https://github.com/ricardobing/excel-to-kpi-dashboard.git
    cd excel-to-kpi-dashboard
    ```
  </Step>

  <Step title="Create and activate virtual environment">
    <CodeGroup>
      ```bash Windows
      python -m venv .venv
      .venv\Scripts\activate
      ```

      ```bash macOS/Linux
      python -m venv .venv
      source .venv/bin/activate
      ```
    </CodeGroup>

    <Note>
      Virtual environments keep your project dependencies isolated from other Python projects on your system.
    </Note>
  </Step>

  <Step title="Install dependencies">
    Install all required packages from the requirements file:

    ```bash
    pip install -r visor_kpi/requirements.txt
    ```

    This installs:
    - **Streamlit** (1.32+) - Multi-page dashboard framework
    - **Pandas** (2.2+) - Data processing and ETL
    - **Plotly** (5.19+) - Interactive visualizations
    - **openpyxl** (3.1.5+) - Excel file handling
    - **Faker** (23.2+) - Mock data generation
    - **pytest** (8.0+) - Testing suite
  </Step>

  <Step title="Generate mock data">
    Navigate to the application directory and run the data generator:

    ```bash
    cd visor_kpi
    python data/mock/generate_mock_data.py
    ```

    This creates `data/raw/mock_data.xlsx` with:
    - **27 months** of historical data (Jan 2023 - Mar 2026)
    - **12 salespeople** with differentiated performance profiles
    - **180 clients** across 4 commercial channels
    - **60 products** in 5 categories
    - **~30,000 transactions** with realistic business patterns

    <Note>
      The script takes approximately 30 seconds to generate the full dataset.
    </Note>
  </Step>

  <Step title="Launch the application">
    Start the Streamlit dashboard:

    ```bash
    streamlit run app.py
    ```

    Your browser will automatically open to `http://localhost:8501` with the dashboard running.

    <Note>
      The dashboard loads in under 3 seconds with the complete dataset.
    </Note>
  </Step>
</Steps>

## What's Next?

Once the application is running, you can:

- **Explore the Executive Summary** - View overall business KPIs and rankings
- **Access Manager View** - See consolidated team performance and trends
- **Check Individual Salesperson Reports** - Each seller can view their own metrics
- **Analyze Clients** - Review Pareto segmentation and identify at-risk accounts
- **Review Products** - Understand product performance and identify low-rotation SKUs
- **Monitor Alerts** - Access the alerts center for proactive business management

To verify everything is working correctly, run the test suite:

```bash
cd visor_kpi
pytest tests/ -v
```

You should see **51 tests passing**, covering:
- Mock dataset validation
- KPI engine calculations
- Pareto analysis logic
- Alert engine rules (ALT001-ALT007)

## Troubleshooting

<Accordion title="Module not found errors">
  If you see `ModuleNotFoundError`, ensure your virtual environment is activated and all dependencies are installed:

  ```bash
  pip install -r visor_kpi/requirements.txt
  ```
</Accordion>

<Accordion title="Port already in use">
  If port 8501 is already in use, specify a different port:

  ```bash
  streamlit run app.py --server.port 8502
  ```
</Accordion>

<Accordion title="Mock data file not found">
  Ensure you're in the `visor_kpi` directory when running the data generator:

  ```bash
  cd visor_kpi
  python data/mock/generate_mock_data.py
  ```

  The file should be created at `visor_kpi/data/raw/mock_data.xlsx`.
</Accordion>

<Accordion title="Python version issues">
  This project requires Python 3.10 or higher. Check your version:

  ```bash
  python --version
  ```

  If you have multiple Python versions installed, you may need to use `python3.10` or `python3.11` explicitly.
</Accordion>
