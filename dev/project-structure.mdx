## Root Files

### `app.py` — Main Entry Point

**Purpose:** Executive summary dashboard (default landing page)  
**Lines:** 312  
**Responsibilities:**
- Configures Streamlit page settings (`st.set_page_config`)
- Loads global CSS from `assets/style.css`
- Renders sidebar filters via `components.filters`
- Displays 4 primary KPI cards: Sales, Team Compliance, Margin, Coverage
- Shows evolution chart, gauge, top-3 rankings, and active alerts


**Lines:** 203

## Key Patterns

### 1. **Separation of Concerns**

```python
# ✅ GOOD: Business logic in src/, UI in components/
kpi = kpi_ventas_periodo(df, df_ant)  # src/kpi_engine.py
render_kpi_card("Ventas", fmt_moneda(kpi["importe_total"]))  # components/kpi_card.py

# ❌ BAD: Mixing calculation and presentation
def render_sales_card(df):
    total = df["importe_neto"].sum()  # DON'T calculate in UI component
    st.metric("Ventas", total)
```

### 2. **Caching Pattern**

```python
# Heavy I/O operations get cached
@st.cache_data(ttl=3600)
def load_data():
    return pd.ExcelFile(DATA_PATH).parse(...)

# Fast filters do NOT get cached (to reflect user input)
def get_filtered_data(fecha_desde, fecha_hasta, ...):
    return get_master_df()[mask]  # Uses cached master DF
```

### 3. **Error Handling**

```python
# Graceful degradation with empty DataFrames
if df.empty:
    return pd.DataFrame()  # or return default dict

# User-friendly error messages
if not check_data_available():
    st.error("⚠️ No data file. Run: python data/mock/generate_mock_data.py")
    st.stop()
```
