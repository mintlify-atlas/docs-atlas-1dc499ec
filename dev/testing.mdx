Visor KPI Comercial includes a comprehensive test suite with **51 passing tests** covering the entire data pipeline, KPI engine, and alert system.

## Test Suite Overview

The project uses **pytest** as the testing framework, with tests organized into three main modules:

| Test Module | Coverage | Tests |
|-------------|----------|-------|
| `test_mock_data.py` | Dataset validation and integrity | 18 tests |
| `test_kpis.py` | KPI calculations and Pareto analysis | 22 tests |
| `test_alerts.py` | Alert engine and business rules | 11 tests |

## Running Tests

<CodeGroup>
```bash Full test suite
cd visor_kpi
```

## Test Data Generation

Tests use the mock dataset generated by:

```bash
cd visor_kpi
python data/mock/generate_mock_data.py
```

This creates `data/raw/mock_data.xlsx` with:
- **27 months** of historical data (Jan 2023 - Mar 2026)
- **~30,000 transactions** with realistic patterns
- **12 vendors** with different performance profiles (star, stable, problematic)
- **180 clients** across 4 commercial channels
- **60 products** in 5 categories

<Info>
If the data file is missing, tests will automatically skip with a message:
```
Archivo de datos no encontrado: data/raw/mock_data.xlsx
Ejecutá: python data/mock/generate_mock_data.py
```
</Info>

## Writing New Tests

### Test Structure

All test files follow this pattern:

```python
import os
import sys
import pytest
import pandas as pd

# Setup path to import project modules
BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
sys.path.insert(0, BASE_DIR)

from config import DATA_PATH
from src.kpi_engine import kpi_ventas_periodo

@pytest.fixture(scope="module")
def data():
    """Loads Excel once for all tests."""
    if not os.path.exists(DATA_PATH):
        pytest.skip("Data file not found")
    xl = pd.ExcelFile(DATA_PATH)
    return {
        "ventas": xl.parse("Ventas"),
        "clientes": xl.parse("Clientes"),
    }

def test_sales_positive(data):
    """Validates all sales amounts are positive."""
    assert (data["ventas"]["importe_neto"] > 0).all()
```

```python
def make_simple_df(n: int = 500, seed: int = 42) -> pd.DataFrame:
```

## Continuous Integration

While no CI/CD is currently configured, tests can be integrated with GitHub Actions:

```yaml .github/workflows/tests.yml
name: Run Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      - run: pip install -r visor_kpi/requirements.txt
      - run: cd visor_kpi && python data/mock/generate_mock_data.py
      - run: cd visor_kpi && pytest tests/ -v
```

## Troubleshooting Tests

<AccordionGroup>
  <Accordion title="ImportError: No module named 'src'" icon="triangle-exclamation">
    **Solution**: Run tests from the `visor_kpi/` directory:
    ```bash
    cd visor_kpi
    pytest tests/ -v
    ```
  </Accordion>

  <Accordion title="Tests skipped: Data file not found" icon="file-excel">
    **Solution**: Generate the mock dataset first:
    ```bash
    cd visor_kpi
    python data/mock/generate_mock_data.py
    ```
  </Accordion>

  <Accordion title="ValueError: cannot convert float NaN to integer" icon="calculator">
    **Solution**: Check for missing data in the Excel. Tests expect complete datasets. Regenerate mock data if corrupted.
  </Accordion>

  <Accordion title="Test failed: Expected 51 tests, got different count" icon="list-check">
    **Solution**: Ensure you're running from the correct directory and have the latest code:
    ```bash
    git pull origin main
    cd visor_kpi
    pytest tests/ --collect-only  # See test count
    ```
  </Accordion>
</AccordionGroup>
