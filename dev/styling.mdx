Visor KPI Comercial uses a combination of CSS (in `assets/style.css`) and Python color configuration (in `config.py`) to create a professional dark-mode interface. This guide shows you how to customize the styling.

## Style Architecture

The styling system has three layers:

1. **CSS Global Styles** (`assets/style.css`) - Controls overall layout, fonts, and Streamlit component overrides
2. **Python Color Constants** (`config.py`) - Defines the color palette used in charts and custom components
3. **Component Styling** (`components/*.py`) - Inline styles in custom UI components

## CSS Structure

The `assets/style.css` file is organized into sections:

```css assets/style.css
/* ── Reset y base ─────────────────────────────────────────── */
* {
    font-family: 'Inter', sans-serif !important;
}

/* ── Ocultar elementos de Streamlit por defecto ──────────── */
#MainMenu { visibility: hidden; }
footer { visibility: hidden; }

/* ── Fondo principal ─────────────────────────────────────── */
.stApp {
    background-color: #0F1923;
}

/* ── Sidebar ─────────────────────────────────────────────── */
[data-testid="stSidebar"] {
    background-color: #1A2535 !important;
    border-right: 1px solid #2D3F55;
}
```

## Changing the Color Scheme

The color palette is defined in two places that must stay synchronized.

### 1. Update config.py Colors

```python config.py
COLORS = {
    # Change these to your brand colors
    "primary":        "#YOUR_PRIMARY_COLOR",
    "primary_light":  "#YOUR_PRIMARY_LIGHT",
    "accent":         "#YOUR_ACCENT_COLOR",
    
    "success":        "#YOUR_SUCCESS_COLOR",
    "warning":        "#YOUR_WARNING_COLOR",
    "danger":         "#YOUR_DANGER_COLOR",
    
    "bg_dark":        "#YOUR_BG_DARK",
    "bg_card":        "#YOUR_BG_CARD",
    "bg_card_hover":  "#YOUR_BG_CARD_HOVER",
    "text_primary":   "#YOUR_TEXT_PRIMARY",
    "text_secondary": "#YOUR_TEXT_SECONDARY",
    "border":         "#YOUR_BORDER_COLOR",
}
```

### 2. Update CSS to Match

Update corresponding colors in `assets/style.css`:

```css assets/style.css
/* Update background colors */
.stApp {
    background-color: #YOUR_BG_DARK;  /* matches COLORS["bg_dark"] */
}

[data-testid="stSidebar"] {
    background-color: #YOUR_BG_CARD !important;  /* matches COLORS["bg_card"] */
}

/* Update card colors */
.kpi-card {
    background: #YOUR_BG_CARD;  /* matches COLORS["bg_card"] */
    border-left: 4px solid #YOUR_PRIMARY_LIGHT;  /* matches COLORS["primary_light"] */
}

.kpi-card:hover {
    background: #YOUR_BG_CARD_HOVER;  /* matches COLORS["bg_card_hover"] */
}
```

<Steps>
  <Step title="Choose your color palette">
    Use a tool like [Coolors](https://coolors.co/) or [Adobe Color](https://color.adobe.com/) to create a cohesive palette.
  </Step>
  
  <Step title="Update config.py first">
    Change all color values in the `COLORS` dictionary.
  </Step>
  
  <Step title="Update CSS to match">
    Search and replace the corresponding hex codes in `assets/style.css`.
  </Step>
  
  <Step title="Test in browser">
    Restart Streamlit and check all pages to ensure colors look correct.
  </Step>
</Steps>

## Customizing KPI Cards

KPI cards are styled using the `.kpi-card` class:

```css assets/style.css
.kpi-card {
    background: #1A2535;
    border-radius: 12px;        /* Rounded corners */
    padding: 20px;              /* Internal spacing */
    border-left: 4px solid #2E5FA3;  /* Left accent border */
    transition: transform 0.2s ease, box-shadow 0.2s ease;
    animation: fadeIn 0.3s ease-in;
}

.kpi-card:hover {
    transform: translateY(-2px);  /* Lift effect on hover */
    box-shadow: 0 8px 24px rgba(0, 0, 0, 0.4);
    background: #243044;
}
```

### Modify Card Appearance

**Make cards more rounded:**
```css
.kpi-card {
    border-radius: 20px;  /* More rounded */
}
```

**Remove hover effect:**
```css
.kpi-card:hover {
    transform: none;  /* No lift */
    background: #1A2535;  /* Same as default */
}
```

**Thicker accent border:**
```css
.kpi-card {
    border-left: 8px solid #2E5FA3;  /* Thicker */
}
```

### KPI Card Elements

KPI cards have three styled sub-elements:

```css assets/style.css
.kpi-label {
```

## Customizing Charts

Chart styling comes from `config.py`'s `PLOTLY_THEME` and is applied in `components/charts.py`:

```python config.py
PLOTLY_THEME = {
    "paper_bgcolor": COLORS["bg_card"],
    "plot_bgcolor":  COLORS["bg_card"],
    "font":          {
        "color": COLORS["text_primary"],
        "family": "Inter, sans-serif",
        "size": 12
    },
    "margin":        dict(l=20, r=20, t=40, b=20),
}
```

### Chart Customization Examples

**Larger chart margins:**
```python
PLOTLY_THEME = {
    # ...
```

**Different font:**
```python
PLOTLY_THEME = {
```

## Changing Fonts

### 1. Import Font in CSS

The dashboard uses Inter from Google Fonts:

```css assets/style.css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap');

* {
    font-family: 'Inter', sans-serif !important;
}
```

To use a different font:

```css
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@300;400;500;700&display=swap');

* {
    font-family: 'Roboto', sans-serif !important;
}
```

### 2. Update Chart Font

Update `PLOTLY_THEME` in `config.py`:

```python
PLOTLY_THEME = {
    # ...
    "font": {
        "color": COLORS["text_primary"],
        "family": "Roboto, sans-serif",  # Match CSS font
        "size": 12
    },
}
```

```css assets/style.css
::-webkit-scrollbar {
    width: 6px;
    height: 6px;
}

    background: #0F1923;  /* Track color */
}

::-webkit-scrollbar-thumb {
    background: #2D3F55;  /* Thumb color */
```

<Note>
Webkit scrollbar styles only work in Chrome, Edge, and Safari. Firefox uses different properties.
</Note>

## Responsive Design

Mobile styles are defined in a media query:

```css assets/style.css
@media (max-width: 768px) {
    .kpi-value {
        font-size: 22px;  /* Smaller on mobile */
    }
    
    .kpi-card {
        padding: 14px;  /* Less padding */
    }
    
    .block-container {
        padding-left: 12px !important;
        padding-right: 12px !important;
    }
}
```

### Add Tablet Breakpoint

```css
@media (max-width: 1024px) and (min-width: 769px) {
    .kpi-value {
        font-size: 26px;
    }
}
```

## Dark/Light Mode Toggle

To add a light mode option:

<Steps>
  <Step title="Define light mode colors in config.py">
    ```python config.py
    COLORS_LIGHT = {
        "primary":        "#1B3A6B",
        "bg_dark":        "#FFFFFF",
        "bg_card":        "#F5F7FA",
        "text_primary":   "#1A202C",
        "text_secondary": "#4A5568",
        # ...
    }
    ```
  </Step>
  
  <Step title="Create conditional CSS classes">
    ```css assets/style.css
    .stApp.light-mode {
        background-color: #FFFFFF;
    }
    
    .light-mode .kpi-card {
        background: #F5F7FA;
        color: #1A202C;
    }
    ```
  </Step>
  
  <Step title="Add toggle in sidebar">
    ```python app.py
    import streamlit as st
    
    theme = st.sidebar.selectbox("Theme", ["Dark", "Light"])
    
    if theme == "Light":
        st.markdown('<style>.stApp { ... light mode CSS ... }</style>',
                    unsafe_allow_html=True)
    ```
  </Step>
</Steps>
