## Overview

All performance thresholds, color schemes, and formatting utilities are centralized in `config.py`. These configurations drive the visual indicators (semaphores) and business logic throughout Visor KPI Comercial.

---

## Performance Thresholds

### Objective Compliance (Cumplimiento)

Thresholds for evaluating sales performance against objectives:

```python
THRESHOLDS = {
    "cumplimiento_ok":      1.00,   # 100% or above
    "cumplimiento_warning": 0.80,   # 80% to 99%
    # Below 0.80 (80%) is considered critical
}
```

**Semaphore Logic:**

| Performance Level | Threshold | Indicator | Color | Label |
|------------------|-----------|-----------|-------|-------|
| **Excellent** | ≥ 100% | 🟢 | `#00C49F` | `success` |
| **Acceptable** | 80% - 99% | 🟡 | `#FFB347` | `warning` |
| **Critical** | < 80% | 🔴 | `#E84855` | `danger` |
| **No Data** | `None` | 🔴 | `#E84855` | `danger` |

<Note>
  These thresholds are used in `kpi_por_vendedor()` to classify salesperson performance and in alert rules (ALT001).
</Note>

**Source:** `config.py:38-42`

### Growth & Decline Thresholds

Thresholds for period-over-period comparisons:

```python
THRESHOLDS = {
    "caida_critica":       -0.20,   # -20% decline triggers alerts
    "crecimiento_notable":  0.15,   # +15% growth is highlighted
}
```

**Business Rules:**

<ParamField path="caida_critica" type="float" default="-0.20">
  **Critical Decline Threshold**
  
  When sales drop by 20% or more vs. previous period:
  - Triggers ALT003 alert for category A customers
  - Highlights vendors with significant underperformance
  - Requires immediate management review
</ParamField>

<ParamField path="crecimiento_notable" type="float" default="0.15">
  **Notable Growth Threshold**
  
  When sales grow by 15% or more vs. previous period:
  - Highlights positive performance in dashboards
  - Used in executive insights
  - May indicate successful initiatives
</ParamField>

**Source:** `config.py:44-45`

## Semaphore Functions

### `get_color_semaforo()`

Returns hex color code based on compliance percentage.

**Function Signature:**
```python
def get_color_semaforo(cumplimiento: float) -> str
```

**Logic:**
```python
if cumplimiento is None:
    return "#E84855"  # Red
if cumplimiento >= 1.00:
    return "#00C49F"  # Green
elif cumplimiento >= 0.80:
    return "#FFB347"  # Orange
return "#E84855"     # Red
```

**Example Usage:**
```python
color = get_color_semaforo(0.92)  # Returns "#FFB347" (orange)
```

**Source:** `config.py:136-148`

## Formatting Functions

### Currency Formatting

#### `fmt_moneda()`

Formats numbers as Argentine-style currency.

**Function Signature:**
```python
def fmt_moneda(valor: float) -> str
```

**Examples:**
```python
fmt_moneda(1284500)     # "$1.284.500"
fmt_moneda(542.75)      # "$542,75"
fmt_moneda(-15000)      # "-$15.000"
fmt_moneda(None)        # "$0"
```

**Logic:**
- Uses `.` as thousands separator
- Uses `,` as decimal separator
- No decimals for values ≥ 1,000
- Two decimals for values < 1,000
- Handles None gracefully

**Source:** `config.py:67-81`

---

### Percentage Formatting

#### `fmt_pct()`

Formats decimal percentages as readable strings.

**Function Signature:**
```python
def fmt_pct(valor: float, decimales: int = 1) -> str
```

**Examples:**
```python
fmt_pct(0.842)          # "84.2%"
fmt_pct(1.05)           # "105.0%"
fmt_pct(0.8, decimales=0)  # "80%"
fmt_pct(None)           # "—"
```

**Parameters:**
- `valor`: Decimal percentage (0.842 = 84.2%)
- `decimales`: Number of decimal places (default: 1)

**Source:** `config.py:84-91`

### Delta Formatting

#### `fmt_delta()`

Formats period-over-period changes with directional arrows.

**Function Signature:**
```python
def fmt_delta(valor: float) -> str
```

**Examples:**
```python
fmt_delta(0.082)       # "▲ +8.2%"
fmt_delta(-0.031)      # "▼ -3.1%"
fmt_delta(0.0004)      # "— 0.0%"
fmt_delta(None)        # "—"
```

**Logic:**
- Uses ▲ for increases > 0.05%
- Uses ▼ for decreases < -0.05%
- Shows "— 0.0%" for changes between -0.05% and +0.05%

**Source:** `config.py:104-116`

---

#### `fmt_delta_html()`

HTML version with color styling for web display.

**Function Signature:**

**Examples:**


**Colors:**
- Green (#00C49F) for positive changes
- Red (#E84855) for negative changes
- Gray (#8899AA) for neutral or None

### `get_periodo_anterior()`

Calculates equivalent comparison period.

**Function Signature:**
```python
def get_periodo_anterior(
    fecha_desde: date,
    fecha_hasta: date
) -> tuple[date, date]
```

**Example:**
```python
# Current period: March 1-15, 2026 (15 days)
desde, hasta = get_periodo_anterior(
    date(2026, 3, 1),
    date(2026, 3, 15)
)
# Returns: (date(2026, 2, 14), date(2026, 2, 28))
# Previous 15 days ending on Feb 28
```

**Logic:**
1. Calculate duration in days
2. Set new end date = old start date - 1
3. Set new start date = new end date - duration

**Used in:** KPI functions for period-over-period comparisons.

**Source:** `config.py:188-193`

## Usage in KPI Engine

### Integration Examples

#### Salesperson Semaphore

```python
from config import THRESHOLDS

def _color_semaforo(cumpl: float | None) -> str:
    if cumpl is None:
        return "#E84855"
    if cumpl >= THRESHOLDS["cumplimiento_ok"]:
        return "#00C49F"
    elif cumpl >= THRESHOLDS["cumplimiento_warning"]:
        return "#FFB347"
    return "#E84855"
```

**Used in:** `kpi_por_vendedor()` to assign performance indicators

#### Executive Insight Generation

```python
from config import fmt_moneda, fmt_pct, fmt_delta

def generar_insight_ejecutivo(kpi_ventas, kpi_vend, alertas_df):
    importe = fmt_moneda(kpi_ventas["importe_total"])
    cumpl = fmt_pct(kpi_ventas["cumplimiento_pct"])
    
    if kpi_ventas.get("importe_vs_anterior"):
        delta_txt = f", {fmt_delta(kpi_ventas['importe_vs_anterior'])} vs período anterior"
    
    return f"El equipo lleva {importe} en ventas{delta_txt}..."
```
